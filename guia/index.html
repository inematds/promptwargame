Prompt de planejamento

Pare de queimar tokens no caminho feliz

Um prompt pronto que transforma qualquer briefing de projeto num war game: fases com suposição otimista/pessimista, modos de falha, critérios de saída verificáveis e uma tabela mestre de riscos — tudo antes de escrever a primeira linha de código.

Começar o guia Ver no GitHub
O que é

Um prompt, não uma ferramenta

É um texto único — em EN e PT-BR — que você cola no início de uma conversa com sua IA de planejamento antes de pedir qualquer execução. Não precisa instalar nada.

🎯 Sem caminho feliz

Cada fase é obrigada a declarar objetivo, suposição otimista, suposição pessimista, modos de falha com correção, critérios de saída verificáveis, caminho alternativo e condição de parada — antes de existir código.

📋 Tabela mestre de riscos

Uma tabela única cobrindo infraestrutura, particularidades da API do modelo, frontend, qualidade de conversa, segurança e loop de aprendizado, cada cenário com detecção e recuperação.

🔁 Loop de autoevolução

Desenha uma revisão semanal que varre as conversas em busca de prompt injection e ajusta o estilo do agente dentro de limites rígidos — nenhuma mudança vai pra produção sem avaliação de regressão e aprovação explícita.

Como funciona

Planeja com modelo forte, executa fase por fase

A lógica por trás do prompt: um esforço alto de raciocínio no planejamento compra um esforço baixo/médio — e muito mais previsível — na execução.

Copiar o prompt→ Preencher os [colchetes]→ Colar como 1ª mensagem→ Plano em fases + tabela mestre→ Dossiê visual em HTML→ Executar fase a fase

Guia de uso · passo a passo

Do prompt ao plano em 5 passos

O prompt foi desenhado para um "cold-start agent" — funciona melhor como a primeira mensagem de uma conversa nova, sem nenhum contexto anterior do projeto.

1

Escolha o idioma e abra o prompt

Os dois arquivos são equivalentes — escolha o do idioma em que você vai conversar com a IA.

cat doc/prompt-war-game-portugues.md  # ou doc/war-game-prompt.md (EN)
2

Preencha os campos entre colchetes

Troque cada [PROJETO], [Claude Haiku], [Supabase Edge Functions], [workshops/serviços] etc. pelos detalhes reais do seu projeto — modelo, infraestrutura, stack, entregáveis esperados.

# antes
Plan a full build of [PROJECT], but do it as a war game.
# depois
Plan a full build of the Prompt Advisers website chat agent, but do it as a war game.
3

Cole como a primeira mensagem

Sem preâmbulo, sem contexto extra — o prompt já pede tudo que a IA precisa saber pra planejar do zero.

nova conversa → colar o prompt preenchido  # 1ª mensagem, sem histórico
4

Cobre os dois entregáveis

Se a IA parar antes de entregar os dois formatos pedidos no prompt, peça explicitamente pelos dois.

1. plano em Markdown (preâmbulo, caminho crítico, stack, estrutura, metodologia)
2. versão visual em HTML com diagramas da arquitetura e de cada fase
5

Execute fase por fase, sem pular

Pare a cada fase, confira os critérios de saída, e só então siga pra próxima. Se bater numa condição de parada, pare de verdade: registre o bloqueio em vez de deixar a IA improvisar uma saída.

fase 0 → checar critérios de saída → fase 1 → ...
Modo massa

Quando o problema não é um projeto, são doze

O prompt acima abre um projeto a fundo. Existe um segundo modo, no mesmo repo, pra quando a pergunta é outra: tenho vários projetos parados — qual eu faço primeiro, qual eu simplifico, qual eu abandono?

⚔️ War Game

Um projeto, a fundo. Responde "como construir isso sem levar porrada no caminho?". Entrega plano em fases, tabela mestre de falhas e dossiê visual em HTML.

🗂️ War Game em Massa

Vários projetos, comparados. Responde "por onde eu começo?". Entrega uma análise por projeto, ranking por impacto/esforço/risco e plano de 30 e 90 dias.

Os dois usam a mesma lógica — ação → reação → contra-ação — e a mesma regra: prever a barreira antes da primeira linha de código. A diferença está em onde o modelo caro é gasto.

Um arquivo por projeto→ Contexto + critérios comuns→ Prompt mestre (modelo forte)→ Rodada de revisão→ Ranking + 30/90 dias→ Executar 1 projeto (modelo barato)
1

Monte a estrutura e escreva um arquivo por projeto

Nunca junte vários num arquivo só — isso mistura contexto e destrói a comparação. Comece pelo briefing mínimo de 5 perguntas; o que você não souber, deixe em branco, não chute.

analise-projetos/
├── projetos/          # projeto-01.md, projeto-02.md, ...
├── resultados/        # a IA escreve aqui
├── contexto-geral.md  # o que vale pra todos
├── criterios.md       # os mesmos critérios pra todos
├── ledger.md          # bloqueios
└── assumptions.md     # suposições, com impacto se erradas
2

Preencha o contexto geral e os critérios

São esses dois arquivos que tornam projetos diferentes comparáveis. Critério comum é o que permite ranking — critério por projeto não permite nada.

# criterios.md — os mesmos pra todos
substância  clareza · problema real · público · valor · diferencial
execução   viabilidade · complexidade · gargalo · simplificação · automação
risco      o que impede · segurança · manutenção · escala
retorno    receita · custo
3

Rode o prompt mestre com o modelo mais capaz

Ele gera uma primeira versão de todos os projetos — sem polir nenhum. Polir o projeto 01 antes de olhar o resto queima o orçamento no primeiro da fila, que raramente é o mais importante.

cat doc/war-game-massa-portugues.md  # ou doc/mass-war-game.md (EN)
4

Rode a rodada de revisão

A primeira rodada quase sempre sai genérica. É a segunda que tira o genérico: caça contradição, risco ignorado, etapa sem validação e ação sem condição de parada.

revisar /resultados → aprofundar war-game → atualizar ledger + suposições
5

Escolha UM projeto e entregue ao modelo barato

Análise pode ser em massa. Execução, não. Leia o resumo-geral.md, escolha o prioritário e entregue o war-game dele com o prompt de execução.

modelo avançado pensa e antecipa
modelo intermediário organiza
modelo barato executa  # seguindo o plano já mastigado

O ledger.md e o assumptions.md são o que impede a análise de virar ficção: sem eles, o modelo inventa o que falta pra conseguir continuar. Faltou informação? Ela vira [PRECISA_DEFINIR: x] no ponto exato onde faria falta.

Quer ver onde isso chega? A saída de exemplo traz um portfólio fictício de 4 projetos analisado de ponta a ponta — uma análise individual completa, o resumo geral com ranking e plano de 30/90 dias, mais o ledger e as suposições. É a referência de "pronto" deste modo.
